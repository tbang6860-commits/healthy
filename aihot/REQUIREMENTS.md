# AI 热点聚合分析平台 — 需求文档

> 版本：v1.0 | 日期：2026-05-05

---

## 1. 项目定位

构建一个**多源实时热点聚合分析平台**，从多个信息源自动抓取热点数据，利用 DeepSeek V4 进行 AI 智能分析，以独特的"脉冲宇宙"主题前端呈现结果。解决单一信息源偏差问题，帮助用户快速掌握全网热点全貌。

## 2. 核心需求

### 2.1 多源数据抓取

| 序号 | 数据源 | 抓取方式 | 说明 |
|------|--------|----------|------|
| R1 | 微博热搜 | Firecrawl API 抓取 | 实时热搜榜单 |
| R2 | 百度热搜 | Firecrawl API 抓取 | 百度实时热点 |
| R3 | 知乎热榜 | Firecrawl API 抓取 | 知乎热门话题 |
| R4 | Twitter 趋势 | twitterapi.io API | 全球 Twitter 趋势 + 热推 |

**约束**：
- 不依赖搜索引擎付费 API，直接抓取目标页面
- 控制请求频率，每 30 分钟一轮
- 多源并行抓取，避免单一源偏差

### 2.2 AI 智能分析

| 序号 | 分析能力 | 说明 |
|------|----------|------|
| R5 | 热点摘要 | 对每个热点生成 50 字以内一句话摘要 |
| R6 | 分类标注 | 自动归类：科技/娱乐/社会/财经/体育/国际/军事/健康/教育/其他 |
| R7 | 情感分析 | 正面/负面/中性，附带情感强度 |
| R8 | 热度打分 | 综合跨源出现频率，给出 0-100 统一热度指数 |
| R9 | 跨源关联 | 识别同一话题在不同平台的讨论，合并关联 |

### 2.3 网页通知

| 序号 | 需求 | 说明 |
|------|------|------|
| R10 | 浏览器通知 | 新热点发现后通过 Web Notification API 弹窗 |
| R11 | 页面内标记 | 新热点卡片显示醒目的红色脉冲动画 |
| R12 | 标题闪烁 | 有新热点时页面 title 前缀 `[新]` 提示 |

### 2.4 定时更新

| 序号 | 需求 | 说明 |
|------|------|------|
| R13 | 30 分钟刷新 | 后端每 30 分钟自动抓取 + 分析一轮 |
| R14 | 手动刷新 | 用户可点击按钮立即触发刷新 |
| R15 | 历史快照 | 保留每轮抓取的数据，支持趋势对比 |

### 2.5 前端展示

| 序号 | 需求 | 说明 |
|------|------|------|
| R16 | Bento Grid 看板 | 大小卡片交替排列，展示 TOP 30 热点 |
| R17 | 分类筛选 | 按类别标签过滤热点 |
| R18 | 来源筛选 | 按数据源过滤（微博/百度/知乎/Twitter） |
| R19 | 热点详情 | 点击展开 AI 分析详情（摘要、情感、来源对比） |
| R20 | 趋势图表 | 过去 24 小时热度变化折线图 |
| R21 | 深色主题 | 玻璃拟态 + 霓虹色点缀 + 脉冲动画 |

### 2.6 Agent Skills（Phase 2）

| 序号 | 需求 | 说明 |
|------|------|------|
| R22 | 热点监控 Skill | 定时检查热点变化，发现异动主动报告 |
| R23 | 热点分析 Skill | 接收用户问题，基于实时热点数据回答 |
| R24 | 热点推送 Skill | 用户可订阅特定类别，定向推送 |

## 3. 非功能需求

| 类别 | 要求 |
|------|------|
| 性能 | API 响应 < 2s，前端首次加载 < 3s |
| 可用性 | 任一数据源故障时，其余源正常服务 |
| 安全 | API Key 仅存后端，前端不暴露 |
| 兼容 | Chrome/Firefox/Edge 最新版 |

## 4. 技术约束

- 后端：Python FastAPI
- AI：DeepSeek V4（官方 API，OpenAI 兼容协议）
- 网页抓取：Firecrawl API（已安装 MCP）
- Twitter：twitterapi.io REST API
- 前端：React 18 + Vite + Tailwind CSS
- 数据存储：SQLite（零配置）
